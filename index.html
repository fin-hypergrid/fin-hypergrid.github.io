fin-hypergrid CDN

Welcome to the Hypergrid CDN. This site hosts build files for Hypergrid core as well as the various Hypergrid plug-ins, data source modules, library modules, mix-ins, etc., whose source repositories are hosted under the fin-hypergrid GitHub organization.

A complete listing of all build files on the CDN along with sample script tags can be found in the CDN index.

Applications use <script> tags to load files from the CDN at run time. In this way, individual modules are loaded onto the client sychronously, one file per <script> tag.

Applications integrated at development time with the Hypergrid npm module (and possibly other npm modules) using a bundler such as Browserify or webpack do not need these build files and should not be loading anything from this CDN at run time.

Editions

All build files are offed in two editions:

  • module-name.js — Unminified versions are for debugging purposes and, when bundled, include a source map.
  • module-name.min.js — Minified versions are smaller and faster and are for use on production.

Usage

  1. Copy and paste <script> tags from the CDN index, as needed, into your application's .html file to have your application load them at run time.
  2. Reference the modules in your in code as illustrated in the next section.

Built-in modules

The following modules are built in to Hypergrid:

  • extend-me
  • fin-hypergrid-field-tools
  • pubsubstar

These modules are implicitly loaded; no <script> tags required! (So skip step 1 above.)

Referencing modules in code

How your modules reference other modules — and how other modules reference your modules — depends on whether or not the modules in question are wrapped up as Hypergrid Client Modules.

Wrapping your modules is easy and highly recommended as doing so allows you to structure your code files like CommonJS modules, which means they are ready as-is for an eventual migration to a bundled build (using a bundler such as Browserify or webpack). See the Client Modules wiki for details.

Hypergrid core

The Hypergrid core object is both a constructor and a namespace and always required. It is referenced as follows:

From raw JavaScript:
var Hypergrid = window.fin.Hypergrid;
From within a Hypergrid Client Module:
var Hypergrid = require('fin-hypergrid');

Hypergrid Client Modules

Most modules whose source repositories are hosted under https://github.com/fin-hypergrid/ are built with the Hypergrid Client Module wrapper and are referenced as follows:

From raw JavaScript:
var myModule = window.fin.Hypergrid.modules['my-module'];
From within another Hypergrid Client Module:
var myModule = require('my-module');

Other modules

All other modules are referenced using a module-dependent global variable. Modules are free to insert themselves into fin.Hypergrid.modules, which would let them be referenced as a Client Module; however they would not themselves be able to reference Hypergrid or Hypergrid Client Modules using require().
